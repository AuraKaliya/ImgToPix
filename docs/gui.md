# GUI 说明

## 设计目标

GUI 现在围绕“先保住物体轮廓，再填颜色”展开：

- 一次导入即可看到前景 `mask`
- 同时预览多个 `16x16` 轮廓候选
- 在选中的轮廓内切换颜色算法
- `32x32` 与 `64x64` 只负责展示，不再独立求细节

## 界面结构

- 左侧改为源图标签页，在 `原图 / Mask / 工作图` 之间切换
- 右上保留 `16x16` 轮廓候选对比
- 右下改为单一主预览，通过尺寸下拉切换 `16x16 / 32x32 / 64x64`

## 补边模式

- `边缘延展`：复制边缘像素向外补边，适合大多数素材
- `镜像补边`：用镜像内容补边，适合主体靠边的图
- `纯色补边`：使用纯色背景补边，适合背景明确的素材

## 轮廓方案

- `coverage-50`：覆盖率过半才保留，轮廓最稳
- `coverage-35`：更宽松，适合保细枝、细腿、细尖角
- `center-hit`：只看块中心，边更利落，但更容易断
- `hybrid`：中心优先，再用覆盖率补细节，适合作为默认推荐

## 颜色算法

- `majority`：按分块多数投票取色，更偏向保住块面
- `median`：按分块中值取色，适合压低锯齿和轻微噪点
- `center`：按分块中心采样，适合网格较整齐的素材

## 导出行为

- 导出的 `16x16` 是当前选中的轮廓方案和颜色算法组合
- 导出的 `32x32` 与 `64x64` 由该 `16x16` 最近邻放大生成
- 主预览只显示一个尺寸，方便在小窗口里工作
- 勾选“同时保存放大预览图”后，会额外保存预览 PNG
