# GUI 说明

## 设计目标

GUI 现在围绕“先做出最好的 `16x16` 母版”展开：

- 一次导入即可同时看到多个 `16x16` 候选方案
- 选中的候选会同步驱动 `16x16`、`32x32`、`64x64` 导出预览
- `32x32` 与 `64x64` 只是母版的最近邻放大结果，保证风格一致

## 界面结构

- 左侧显示原图与补成正方形后的工作图
- 右上显示多个 `16x16` 母版候选
- 右下显示当前选中候选对应的 `16x16`、`32x32`、`64x64`

## 参数说明

- `边缘延展`：复制边缘像素向外补边，适合大多数素材
- `镜像补边`：用镜像内容补边，适合主体靠边的图
- `纯色补边`：使用纯色背景补边，适合背景明确的素材

## 母版算法

- `majority`：按分块多数投票取色，更偏向保住大轮廓和块面
- `median`：按分块中值取色，适合压低锯齿和轻微噪点
- `center`：按分块中心采样，适合本身网格就较整齐的素材

## 使用建议

- 如果想要最稳妥的结果，先看 `majority`
- 如果边缘有点脏，可以试 `median`
- 如果原图本来就很接近整数倍像素放大，可以试 `center`

## 导出行为

- 导出的 `16x16` 是当前选中的母版
- 导出的 `32x32` 与 `64x64` 均由该母版使用最近邻放大生成
- 勾选“同时保存放大预览图”后，会额外保存预览 PNG
